# Books

Essential reading for software architecture mastery.

## Clean Architecture

| Book | Author | Description |
|------|--------|-------------|
| **Clean Architecture** | Robert C. Martin (Uncle Bob) | The definitive guide to software structure, covering SOLID principles, component design, and architectural boundaries. Essential foundation for any architect. |
| **Clean Code** | Robert C. Martin | Writing maintainable, readable code that forms the basis of good architecture. Focus on meaningful names, functions, comments, and formatting. |

## Domain-Driven Design

| Book | Author | Description |
|------|--------|-------------|
| **Domain-Driven Design** | Eric Evans | The seminal work on DDD. Covers strategic design (bounded contexts, ubiquitous language) and tactical design (entities, value objects, aggregates, domain events). Dense but essential. |
| **Implementing Domain-Driven Design** | Vaughn Vernon | Practical implementation guide for DDD. Bridges the gap between Evans' theory and real-world code. Covers aggregates, repositories, event sourcing. |
| **Domain-Driven Design Distilled** | Vaughn Vernon | Condensed version ideal for teams adopting DDD. Focuses on the most impactful patterns: bounded contexts, microservices, and aggregates. |

## Architecture Patterns

| Book | Author | Description |
|------|--------|-------------|
| **Patterns of Enterprise Application Architecture** | Martin Fowler | Catalog of enterprise patterns: domain logic patterns, data source architectural patterns, object-relational behavioral patterns. The PoEAA bible. |
| **Enterprise Integration Patterns** | Gregor Hohpe & Bobby Woolf | Comprehensive guide to asynchronous messaging patterns for enterprise integration. Essential for event-driven architectures. |
| **Building Evolutionary Architectures** | Neal Ford et al. | Fitness functions, architectural coupling, and incremental changes. Focus on supporting evolutionary change over time. |

## Microservices

| Book | Author | Description |
|------|--------|-------------|
| **Building Microservices** | Sam Newman | Practical microservices guide covering service decomposition, communication patterns, deployment strategies, and testing. |
| **Microservices Patterns** | Chris Richardson | Deep dive into microservices patterns: decompose by business capability, saga for transactions, CQRS, event sourcing. |

## Object-Oriented Design

| Book | Author | Description |
|------|--------|-------------|
| **Agile Software Development** | Robert C. Martin | Introduces SOLID principles and principles of package design. Shows how agile practices support architecture. |
| **Design Patterns** | Gang of Four (GoF) | Classic catalog of 23 design patterns. Foundation for understanding reusable object-oriented design. |

## Related Topics

| Book | Author | Description |
|------|--------|-------------|
| **Refactoring** | Martin Fowler | Systematic approach to improving code structure. Essential for maintaining architectural quality over time. |
| **The Phoenix Project** | Gene Kim et al. | Novel about DevOps and IT operations. Illustrates how architecture affects organizational flow. |
