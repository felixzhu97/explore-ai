# C4 Model

A method for software architecture documentation using four levels of abstraction: Context, Container, Component, and Code. Each level serves a different audience and purpose.

## Reference Files

Detailed content for each aspect of the C4 model:

- [C4 Four Levels](./c4-four-levels.md) — The four abstraction levels with ASCII diagrams showing System Context, Container, Component, and Code
- [C4 PlantUML Templates](./c4-plantuml-templates.md) — PlantUML code for generating C4 diagrams, including Context, Container, Component, and Deployment templates
- [C4 Documentation Best Practices](./c4-documentation-best-practices.md) — Guidelines for keeping diagrams accurate, matching audience to level, and integrating with CI/CD
