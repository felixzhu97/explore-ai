# C4 Four Levels

The C4 model defines four levels of abstraction for software architecture documentation, each serving a different audience and purpose.

## When to Use

Use the C4 model when you need to communicate software architecture to different audiences вҖ” from business stakeholders who need a high-level overview, to developers who need implementation details. Start at Level 1 (Context) and drill down only as needed.

## The Four Levels

### Level 1: System Context

Shows how software systems relate to each other and the people who use them. This is the highest level of abstraction.

**Questions to answer:**
- What is the system?
- Who uses it?
- What external systems does it integrate with?

## Real Implementation Reference

In a real Java/Spring Boot project, the C4 Component level maps directly to the package structure:

```
src/main/java/com/ai/
в””в”Җв”Җ interface/
    в””в”Җв”Җ api/
        в””в”Җв”Җ AccountController.java     # Controller вҶ’ adapters
    в””в”Җв”Җ service/
        в””в”Җв”Җ AccountService.java       # Use case orchestration
    в””в”Җв”Җ domain/
        в””в”Җв”Җ model/
            в””в”Җв”Җ Account.java          # Domain entity
        в””в”Җв”Җ repository/
            в””в”Җв”Җ AccountRepository.java # Domain interface
    в””в”Җв”Җ infrastructure/
        в””в”Җв”Җ persistence/
            в””в”Җв”Җ JpaAccountRepository.java # Infrastructure adapter
```
